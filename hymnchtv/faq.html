hymnchtv UI design & Implementation
[20201229] hymnchtv UI design & Implementation { Content }
In hymnchtv UI design, user interaction with the application uses mainly android device touch screen and gestures to navigate or access to other menu. Generally hymnchtv uses the following gestures and click actions to access to other part of the menus.
  • Pull Down Menu: Each menu is usually contains other user selectable options. These options are accessed via tool icons shown in the navigation bar, or pull down for menu overflow options. On some older android device, there is a dedicated Menu button.
  • Context Menu: All hymnchtv context menus are accessed via long click/press on the item. Long press on hymn content will pop up the context menu e.g. to show/hide the media player ui.
  • ViewPager: hymnchtv uses android ViewPager concept to present user selected hymn content for display. User slides up/down to place hymn score or lyrics into screen view area; or left/right to access to other hymn number contents of the similar category.

[20210216] hymnchtv main user interface { Content }
This is the main hymnchtv UI that is shown when user launches the hymnchtv application. From this main UI window, you can access to all the hymnchtv features implementation, pop-up menu and the main pull down menu.
  • Hymn Selection: User can input the required hymn no using the numerical and 附/fu keypad buttons; Then select the HymnType [儿诗|新颂|补充|大本] to enter the hymn lyrics display and media playback UI page.
  • 目录 Hymn Categories: The user desired hymn title can also be entered via the Hymn Category spinner options. Every HymnType are sub-categorised according to:
    诗歌类别: Hymn content category types as given in the various printed hymn books;
    笔画索引: The number of strokes of the first character in the hymn title;
    拼音索引: The first alpha letter of the hymn title first character's PinYin;
    英中对照: Cross reference table from English to Chinese hymn number.
  • 内容搜索: Hymn Content Search: Allow searching the contents for all the HymnType with the search string entered by the user. Hymnchtv supports only search string in simplified Chinese. For special character i.e. ‘祂’, user can input ‘他’, then long press the Content Search button to convert ‘他’➡‘祂’.
  • 附 Fu: Supplement song titles are only available in 大本诗歌. Press 附 follows by number; then press DaBu to enter the selection.
  • Del: Press to Del button to erase the number entry, before enter a new hymn number.
Main UI pull down menu
The main UI pull down menu is acessed via the oveflow icon on the top-right corner of the main UI. It contains the following user selectable options:
  • 墙纸: This option allows user to select one of the 15 hymnchtv predefined images to use as the main UI wallpaper. The sub-menu also contains the option to allow user to define and use own image as the wallpaper. Watch demo video for details..

  • 设置主面版字体颜色: The main UI font color can be changed via this option; there are 8 predefined colors for user selection.
  • 歌词文字最大比例: The hymn lyrics display text size can be changed i.e. toggle via 2-fingers double click, between the predefined default and this user defined scaling factor. User may also use pinch-to-zoom gesture to change he lyrics display text size.
  • 用户定义媒体设置: User defined media configuration setup. See below for details.
  • 网上帮助: Click on option will display this online help page/
  • 关于: About the basic info of the hymnchtv, including install apk version and history.
  • 退出: This will closed the hymnchtv app and exit. The BackKey only close the main UI.

[20220105] Hymn media player UI interface, control and playback operations { Content }
Beside showing the hymn lyrics of the user selected HymnNo, Hymnchtv allows the playback of the media content of the user selected HymnNo. Below lists the various hymn media available for the playback.
  • 搜索: Search the relevant contents for the user selected HymnNo per the button actions below:
    • singleTap to perform hymn info search via qq.com; Will show the QQ main page if none found. You may then navigate to the required hymn title for viewing, or share the link with hymnchtv for later retrieval. See also User defined media configuration
    • longPress to perform hymn info search via youtube.com
    • DoubleTap to perform hymn info search via google.com
    • Note: The user selected HymnNo info is also copied to the clipboard. You may paste this clipboard text into any of your preferred search engine to perform addtional content searching.
  • 媒体: User defined media contents; these may include video, audio and youtube/internet media url links. Pending on the media type for video content, the player used for playback can either be exoplayer or youtube player as shown on the right. The button text greyed out when there is no user media content defined in the database.
    Note: LongPress on this button to show QQ-info main page; allowing user to enter QQ main page in case the 搜索 failed.
  • 伴奏: Hymn accompany music only media playback. The playback media can be using embedded midi files or downloaded from the predefined site. The button text greyed out when there is no user media content defined in the database.
  • 教唱: Hymn singing practice. Currently this is only available for 大本诗歌 DaBen HymnType. The button text greyed out when there is no user media content defined in the database.
  • 唱诗: Hymn singing music with lyrics. The button text greyed out when there is no user media content defined in the database.
  • Repeat/Loop Count: Disable/Enable repeat playback with user defined loop count.
  • Playback speed: Variable playback speed. The available playback speed varies from 0.6x~1.4x of the orignal hymn tempo.
Each hymn media button also provides the availabilty staus/indication of the local media content for playback. The button text is greyed when there is no local media file and required to be downloaded online when playback starts.

For hymn media playback: user first selects the required MediaType, the desired repeat/loop count and the playback speed; then followed by:
a. Click the play button to start;
b. Click again to pause the playback;
c. Click again to resume playback;
d. Long click to stop the playback and return to start of the hymn media.
User may scroll left/right to a different HymnNo lyrics while the hymn playback is in progress. The current playing hymn will continue until the end, without any disruption. If user wants to play the newly selected HymnNo, he/she must manually stop the playback, or wait until the current hymn playback ends. The media player UI info will then be updated to the user newly selected when playback ended.
Except for the 媒体 media type selection, if hymnchtv cannot find the required media file in the local directory for the playback; it will attempt to download the required media content from a predefined url link location. This is internally set to https://heavenlyfood.cn/hymns/ currently. The playback will start when the medium content downloading is completed. The downloading process will only perform once for each of the HymnNo; all the downloaded media files are stored in the local storage for next time use.
If the user closes the current downloading UI by click the X, the downloading process will continue in backgroud until completion. User may select another media type to playback; only returns to the the downloading hymn when it has completed.
[20201231] Hymn media storage and selection algorithm for playback { Content }
The screenshot on the right shows the directory structure for hymnchtv, where all the media files for each of the HymnType and MediaType, either downloaded or added by user are saved to. The main hymnchtv directory are placed below the android Download directory. Each of the hymnchtv sub-directories gets created whenever user initiates playback the respective HymnType/MediaType on the very first instance.
  • hymn_bb: 补充本 HymnType sub-directory
  • hymn_db: 大本诗歌 HymnType sub-directory
  • hymn_er: 儿童诗歌 HymnType sub-directory
  • hymn_xb: 新歌颂咏 HymnType sub-directory
  • import_export: A directory where all the generated export files are saved to. User is responsible to upkeep this directory. A copy of the user selected external export/import file will also be kept here.
  • tmp: A temporary directory location to keep user selected hymnchtv media files during user defined media configuration. All files are deleted after user committed the changes.
  • wallpaper: A directory where all the user defined wallpaper are kept
Under each of the HymnType directories, it contains the sub-directories for each of the four hymnchtv MediaTypes as listed below. All the media files in these locations may be downloaded automatically by hymnchtv from the predefined site location i.e. https://heavenlyfood.cn/hymns/, or added in by the user during user defined media configuration 用户定义的媒体内容 process.
  • media_banzou: 伴奏 MediaType sub-directory; the 伴奏 playback will use either the app embedded midi files, or downloaded accompaniment music in this directory if available.
  • media_changshi: 唱诗 MediaType sub-directory; containing the downloaded files for the 唱诗 playback.
  • media_jiachang: 教唱 MediaType sub-directory; currently, the media are only available for 大本诗歌.
  • media_media: 媒体 MediaType sub-directory; saving location for the media files added in by user during
    User defined media configuration process.
Media content selection algorithm for playback
When user clicks on the play button to playback the selected MediaType content; hymnchtv uses the following algorithms to select the actual media contents for playback.
  1. It first checks the database, for a user defined media record per the selected HymnType/MediaType; and uses the specified media content or url link for playback if available; else
  2. It checks for a previously downloaded local media file per current selected HymnType/MediaType; if none is found, it proceeds and attempts to download from the pre-defined internet site.
  3. If neither of the above attempts has the required media file; starting from left to right per the [伴奏|教唱|唱诗] radio buttons layout, it uses the next inline MediaType as fallback, the previous step #2 is repeated until the end of the radio button list.
Note:If desire, user may overwrite any of the previous downloaded media file for the respective HymnType/MediaType with his own media content. Alternately, use the User define media configuration method as described in the next section.
[20220105] User defined media configuration { Content }
Hymnchtv allows user to define a given media content playback to be associated with a HymnType/MediaType/HymnNo. The media content can either be a local saved medium file, an url link of an online audio/video media address, including the youtube shared link; the url link can also be a downloadable link of an audio media which can either be in mp3 or midi format. Hymnchtv supports both the video, and audio media playback. The user defined media is primarily meant for the 媒体 MediaType, however you may also apply it to other MediaType [伴奏|教唱|唱诗] if desired. Please follow below steps to add a user defined media to associate it with a given HymnType/MediaType/HymnNo.
Local media file association
  • Decide which HymnType/MediaType the media content should be associated with.
  • Copy the media file to your android device SDCard location. It is recommended that the file save location is the actual HymnType/MediaType sub-directory, so there is only a single copy exists on your device.
  • Use the Folder explorer to select and input the media filepath. Due to android file access security, the actual filepath in the 媒体链接 field entry may be a tmp path.
  • [诗歌类别|媒体类型]: Using the two given android spinners in the UI, select the associated HymnType/MediaType for the selected media content.
  • 诗歌编号: Enter the Hymn Number; check the 附歌 option if applicable.
  • Click the 加添 button to add/save the media record into the database.
    • This newly added media content will be used for the playback when user selects its associated hymn data i.e. HymnType/MediaType/HymnNo in the lyrics display window.
  • 删除: Use this button to delete media record that is no further required. If you re-assign/move a medium content to a different HymnType, user must peform the re-assignment, then followed by a deletion of the old media record of the previous HymnType manually; else the old media record will remain in the old HymnType database. This is not necessary if you are just modified the HymnNo of a media record within the same HymnType category.
  • 播放: After you have saved the new media record, or selected an item from the media record list; click the 播放 button to play the media content. The playback will be in the hymn score content window for user confirmation.
  • 退出: Click the 退出 button to close the media record listing window, or exit the user defined media configuration setup.
Follow the steps given below, if the desired medium content to associate with a given HymnType/MediaType is a medium from youtube site or a linked item from website. Below gives detailed descriptions on how to associate for each type of the content sources:

  • Using a browser e.g. Google, open the site where your desired media contents reside.
  • Click and hold on the desired item object; a pop up window appears showing all the available options for the selected item.

  • Proceed to select and input all other fields in the media configuration window i.e. HymnType/MediaType/HymnNo etc to associate with the added web url link.

    自建导出 process and operation:
    One can use the 自建导出 to auto-create the export file for a batch media contents import; instead of doing the media record one at a time. Follow the steps given below to perform this operation. For this to work properly and effectively, the media file should contain the actual HymnNo as part of the filename.
    • Copy all the user desired media files into their associated HymnType/MediaType sub-directory. Please refer to Hymn media storage and selection algorithm for playback on the directory structure used by hymnchtv. It is recommended that you use the defined directory structure, to ease this process.
    • From the user defined media configuration page, select the HymnType, to specify to the auto generation routine the location where all the media files are being saved.
    • Click the 自建导出 button to proceed with the auto export file creation process. Basing on the given HymnType, and the HymnNo embedded in the media file name, each media record is created and added to the export text file.
    • After the 自建导出 operation, the actual created filename/path is shown in the 数据库导入文件 field and a RichText Editor is launched with the file content inserted for user verification and correction. After the modifications, click the 确认 to save the changes
    • 覆盖现有记录 Check this option if you want hymnchtv to overwrite any existing media record in the database during the database import process.
    • Click the 数据库导入 button to import the file content into the hymnchtv database.
    • Click the 数库列项 button to check all the media records that have been imported into the hymnchtv database properly.
    • If there is any error found in 数库列项, you can make minor correction, using either one of the following methods:
      * click on the error item in the list and use the media record UI to make changes before saving the updated record; or
      * return to the exported file using the embedded RichText Editor to do the correction. Then perform the 数据库导入 process with 覆盖现有记录 option checked.
    Export file media record string format:
    Each media record string is consisted of the following parameters separated by commas:
    • HymnType: This is the hymnType with possible values of [hymn_db|hymn_bb|hymn_xb|hymn_er]. Note the text are in lower case.
    • HymnNo: The associated hymnNo for the given HymnType's media content. For 大本诗歌附歌 (Fu), the actual hymnNo stored in database starts with 781 for 附1 respectively, this is shown in the exported file above. User however may use '1' for 附1 when creating the export file. Hymnchtv can accept either one.
    • Fu: 大本诗歌附歌 flag; Set this to '1' if the hymnNo denotes 附歌 else set it to '0' e.g.: hymn_db,781,1,HYMN_MEDIA,null,/storage/emulated/0/
      Download/hymnchtv/hymn_db/media_media/ChHymns-c001.mp3
    • MediaType: The hymn mediaType with all the possible string text values specified in upper case i.e.: [HYMN_MEDIA|HYMN_BANZOU|HYMN_JIAOCHANG |HYMN_CHANGSHI];
    • Media Url: This parameter specifies the actual web/youtube url link. Otherwise it is set to null if the associated media content is a local media file. e.g.:
      hymn_bb,015,0,HYMN_MEDIA,
      https://youtu.be/432RvdcGaM4,null
    • Media FilePath: This parameter specifies the full path for the local stored medium file. This field should be set to 'null' if the media content is referring to an url link. e.g.:
      hymn_db,616,0,HYMN_MEDIA,null,/storage/emulated/0/
      Download/hymnchtv/hymn_db/media_media/ChHymns-0616.mp3
